# Maixsense A010 Python API

This package exposes Maixsense A010 control and data extraction without using the COMTool GUI.
Full Disclosure, this package was entirely generated by an LLM given the original COMTool package as context to reverse engineer.

## Coverage

The API includes wrappers for the A010 controls used in `COMTool/plugins/graph_widget_metasenselite.py`:

- `AT+ISP`
- `AT+DISP`
- `AT+ANTIMMI`
- `AT+BINN`
- `AT+BAUD`
- `AT+UNIT`
- `AT+FPS`
- `AT+AE`
- `AT+EV`
- arbitrary raw AT commands via `send_command(...)`

It also exposes full USB frame metadata + payload parsing and HTTP `/set_cfg`, `/getdeep`, `/getinfo` support from the provided scripts.

## USB quick start

```python
from maixsense_a010_api import MaixsenseA010USB

ports = MaixsenseA010USB.list_serial_ports()
print(ports)

with MaixsenseA010USB(port="/dev/ttyUSB0", baudrate=921600) as sensor:
    sensor.set_display(usb=True, lcd=True, uart=False)
    sensor.set_fps(19)
    sensor.set_binning(1)
    sensor.set_unit(0)

    for frame in sensor.iter_frames(timeout_s=5):
        depth_idx = frame.depth_index_image()
        depth_mm = frame.distance_mm_image(unit=0)
        print(frame.frame_id, frame.shape, frame.sensor_temp, frame.driver_temp)
```

## HTTP quick start

```python
from maixsense_a010_api import MaixsenseA010HTTP, FrameConfig

client = MaixsenseA010HTTP("192.168.233.1", 80)
client.set_config(FrameConfig(trigger_mode=1, deep_mode=0, ir_mode=0, status_mode=2, rgb_mode=1))
frame = client.get_frame()
print(frame.frame_id, frame.depth.shape if frame.depth is not None else None)
```

## Notes

- USB frame payload is parsed as a depth-index image (`uint8`) matching the COMTool MetaSenseLite parser.
- Use `send_command("AT+...\r")` for firmware commands not wrapped yet.
- For baud changes, use `apply_baud_index(index)` to switch both device and host serial baud consistently.

## High-level convenience script

Run as module:

```bash
python -m maixsense_a010_api.cli --port /dev/ttyUSB0 --baud 921600 --seconds 10 --fps 19 --save-npy --save-csv --output-dir ./capture
```

Optional PNG export (requires Pillow):

```bash
python -m maixsense_a010_api.cli --port /dev/ttyUSB0 --save-png --output-dir ./capture_png
```

The convenience script can:

- connect to USB serial
- apply sensor config (`ISP`, `DISP`, `ANTIMMI`, `BINN`, `UNIT`, `FPS`, `AE`, `EV`)
- run additional raw commands (`--cmd "AT+..."`)
- stream frames and print per-frame live telemetry
- record data to `.npy`, `.png`, and `frames.csv`

## Testing

From workspace root:

```bash
python3 -m unittest discover -s tof/tests -p 'test_*.py' -t tof -v
```

This runs deterministic unit tests (mocked serial transport, parser validation, command emission checks).

To run the real-hardware integration test (actual sensor connection + one-frame extraction):

```bash
A010_TEST_PORT=/dev/sipeed A010_TEST_BAUD=921600 python3 -m unittest tof.tests.test_integration_hardware -v
```

If `A010_TEST_PORT` is not set, hardware test is skipped automatically.
